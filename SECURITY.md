# Security Policy

## Supported version

| Version | Supported |
| --- | --- |
| `0.1.x` | Yes |

## Reporting a vulnerability

Do **not** post exploit details, secrets, or production attack paths in a public issue.

Use one of these routes instead:

1. Open a private vulnerability report through GitHub, if the repository Security tab exposes that option.
2. If private reporting is not available, contact the maintainer through the public GitHub profile:
   - [https://github.com/AI-Nikitka93](https://github.com/AI-Nikitka93)
   - public email listed on the profile at the time of review: `nikitka9318@gmail.com`

Include:

- affected route or component
- impact
- reproduction steps
- whether secrets or third-party credentials are involved

## Response target

Best effort:

- initial acknowledgment within 5 business days
- fix or mitigation path after the issue is reproduced
