## Claimed But Not Proven
- Обещание стабильных `5` публикаций в день пока не переподтверждено этим аудитом; `docs/STATE.md` и `docs/EXEC_PLAN.md` сами говорят, что long-run observation еще продолжается.
- Live Telegram delivery не проверялась свежим сообщением в рамках этого запуска.
- Preview run не был свежо вызван, потому что текущая локальная shell-сессия не имела прямой инъекции `CRON_SECRET`.
- Health surface есть, но его реальная полезность как dependency-health индикатора не доказана.

## Missing Expected Capabilities
- В публичном посте нет source attribution: runtime `MiroPost` несет `source`, но схема `posts`, mapping и UI его не сохраняют и не показывают.
- Sports остается first-class category в навигации и schedule, но текущий public surface пуст по этой категории и сам проект признает sports ingest нестабильным.
- У reader surface нет site-level entrypoint в Telegram, хотя README позиционирует Telegram как отдельную ключевую public surface.
- Нет reader-facing provenance для fallback mode: опубликованный пост не маркируется как normal generation vs `editorial_fallback` vs `timeout_fallback`.

## Operator / Maintenance Capabilities
- `cron.yml` вызывает production `/api/cron` по частому polling-расписанию и поддерживает ручной запуск с `strategy/topic`.
- Есть basic ops-alerting на skipped slots и Telegram delivery warning через отдельный alert bot.
- `app/api/revalidate/route.ts` дает защищенный ручной cache reset для public surfaces.
- `app/api/health/route.ts` дает быстрый health ping для smoke/runbook.
- `cd.yml` строит prebuilt Vercel deploy и запускает `pre-launch-check.sh` по production target.

2. Либо восстановить sports lane до устойчивого public output, либо временно убрать `Sports` из first-class surface, пока он не перестанет быть пустой категорией. Иначе обещанный topic surface расходится с текущей реальностью. Сложность: `💛 Средне`

## Functional Summary
- **Functional status:** узкий, но реальный автономный publishing surface уже есть: сайт читает из Supabase, cron/publish loop богатый по логике, RSS и operator workflows на месте.
- **Главный подтверждённый capability surface:** feed/archive/detail/RSS для читателя плюс защищенный cron/scheduler/revalidate contour для оператора.
- **Главный functional mismatch:** продукт обещает четырехтемный автономный дневник, но текущая публичная реальность сильно перекошена в `Markets`, а sports surface в этом аудите оказался пустым.
- **Главный следующий шаг:** сначала сделать queryable run-history + source/provenance persistence, затем либо починить sports ingress, либо честно сузить public promise.
