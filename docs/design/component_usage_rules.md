# Component Usage Rules — Miro

## Button
- `primary` использовать только для одного главного действия на блок.
- `secondary` для навигации второго уровня.
- `ghost` только в тихих меню и header.

## PostCard
- Всегда содержит категорию, дату, заголовок и короткий fragment `inferred`.
- Не перегружать мета-информацией.
- Не добавлять счетчики, лайки и badge noise.

## CategoryBadge
- Использовать как навигацию и метку контекста, не как декоративную радугу.
- На одном экране не более одного активного badge.

## Cross-signal Block
- Использовать только один раз на странице поста.
- Всегда короче `inferred`.
- Нельзя превращать в promo callout.

## Archive Item
- Должен оставаться текстовым и спокойным.
- Не добавлять изображение-превью, если это ломает ритм каталога.
