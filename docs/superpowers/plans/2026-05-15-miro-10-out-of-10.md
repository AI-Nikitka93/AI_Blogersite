# Miro 10/10 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn Miro from a technically working autonomous blog into a trustworthy, writer-led AI publication where every public post is one verified story, written by the primary model, and presented with clear source trust.

**Architecture:** The publish path must move from "publish via fallback when possible" to "publish only verified writer output." Source ingestion must produce a single verified story object before generation. UI must expose source trust and reduce repeated explanation blocks.

**Tech Stack:** Next.js App Router, TypeScript, Supabase, Vercel/GitHub Actions cron, Groq, NVIDIA NIM, RSS/API connectors, local eval scripts.

---

## Council Verdict

Current score: `6/10`.

The site is alive and protected better than before, but it is not yet a premium AI blogger. The blockers are editorial architecture, source truth, fallback behavior, and product trust, not raw model size.

## Definition Of 10/10

- Every public post is one verified story with server-owned source metadata.
- Zero public Tech/Sports/World fallback posts.
- At least one live canary proves `openai/gpt-oss-120b` writer-only success.
- Last 20 posts have no mixed-story observed facts, no ellipsis titles, no boilerplate endings, no source-less metadata.
- One full production day: 5 scheduled windows, clear result for each, fallback rate `0`, editorial skips explained.
- UI exposes source trust and gives a clean reading path on mobile and desktop.
